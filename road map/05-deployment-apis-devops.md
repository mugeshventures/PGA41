# 05. Deployment, APIs, and DevOps

## Production deployment

### Key objectives
- Build robust inference services for AI models
- Containerize applications with Docker
- Deploy to cloud platforms or edge environments

### Common patterns
- REST APIs or gRPC endpoints for model inference
- Serverless inference for small workloads
- Microservices and model serving frameworks

## Tools and frameworks

- FastAPI, Flask, and Django for API layers
- Docker, Kubernetes, and Helm for deployment
- AWS SageMaker, Google Vertex AI, Azure ML, or self-hosted inference

## APIs and integrations

### What AI Engineers must know
- OpenAI API and equivalents from Anthropic, Google, Cohere
- Hugging Face Inference API and Transformers.js
- Managing API keys, quotas, and cost controls

## Observability and reliability

- Logging, metrics, and tracing for AI workloads
- Model performance tracking and drift detection
- Alerts and health checks for inference services
